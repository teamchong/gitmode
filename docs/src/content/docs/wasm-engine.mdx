---
title: WASM Engine
description: Zig-compiled WASM module for git binary operations.
---

The core binary operations (SHA-1, zlib via libdeflate, packfiles, deltas) run in Zig-compiled WASM modules with SIMD128 acceleration. Two modules are available:

| Module | Size | Use case |
|--------|------|----------|
| **Server** (`gitmode.wasm`) | 865KB | Full git server — all operations including libgit2 |
| **Core** (`gitmode-core.wasm`) | 83KB | Client-side — SHA-1, zlib, delta only (no packfile/libgit2) |

## Build Pipeline

```
Zig 0.15 (ReleaseSmall)
  → wasm-metadce (remove unused exports)
  → wasm-opt -Oz (size optimization)
  → 865KB server / 83KB core
```

Build commands:

```bash
pnpm run build:wasm       # Server module only
pnpm run build:wasm-core  # Core module only
pnpm run build:wasm-all   # Both modules
```

## Exports

The WASM module exports 28 functions organized by category:

### Memory Management

| Export | Description |
|--------|-------------|
| `memory` | Linear memory |
| `alloc` | Allocate bytes in WASM heap |
| `resetHeap` | Reset heap (arena allocator) |
| `getHeapUsed` | Get current heap usage |

### SHA-1

| Export | Description |
|--------|-------------|
| `sha1_hash` | Hash raw bytes |
| `sha1_hash_object` | Hash with git object header (e.g., `blob 5\0hello`) |

### Zlib

| Export | Description |
|--------|-------------|
| `zlib_inflate` | Decompress zlib data |
| `zlib_inflate_tracked` | Decompress with size tracking |
| `zlib_deflate` | Compress data |

### Object Parsing

| Export | Description |
|--------|-------------|
| `parse_object_header` | Parse `type size\0` header |

| Export | Description |
|--------|-------------|

### Delta Encoding

| Export | Description |
|--------|-------------|

### Pkt-Line

## Size Optimization

The server WASM binary is 865KB after optimization:

1. **wasm-metadce** — Dead code elimination based on an export graph. Only 28 exports are kept; all unreachable functions are removed. This reduced function count from ~3000+ to 1272.

2. **wasm-opt -Oz** — Binaryen's aggressive size optimizer. Applies code folding, constant propagation, dead argument elimination, and instruction combining.

Both tools require feature flags for the WASM features used:

```bash
--enable-simd --enable-bulk-memory --enable-nontrapping-float-to-int
```

## Zig Source

The WASM source lives in `wasm/` with two build targets:

- `zig build wasm` — server module (`main.zig`, includes libgit2)
- `zig build core` — core module (`main_core.zig`, SHA-1/zlib/delta only)

Key build settings:

- `entry = .disabled` — no `_start` entry point (library mode)
- `rdynamic = true` — export all public functions (required in Zig 0.15)
- `stack_size = 4MB` — sufficient for deep tree recursion and large packfiles

### Zlib via libdeflate

Zlib operations use vendored [libdeflate 1.25](https://github.com/ebiggers/libdeflate) (`wasm/vendor/libdeflate/`), imported via Zig's `@cImport`. This replaced Zig's `std.compress.flate` which crashed on certain inputs. libdeflate is purpose-built for in-memory inflate/deflate and consistently faster than general-purpose zlib implementations.
